## T013.4 disclosed design choices

- Fixture host trees are materialized at runtime in `mktemp` (no committed fixture dirs).
- The suite scripts are mode `0644` and are invoked via `bash <script>` (the way `run_m8_suite()` calls
  them, `bin/dmc:195`); a bare `./script` returns 126 (no execute bit) — a harmless permission bit, not
  a functional defect, since every real caller uses `bash`.
- Byte-clean assertions prune the empty-scaffold directories before `diff -r` (empty dirs carry no
  bytes and git does not track them).
- Merge byte-restoration is canonical-form only: fixtures with pre-existing merge targets are
  constructed in DMC-canonical JSON so the merge→strip round-trip is byte-exact; a non-canonical host
  file is honestly documented as SEMANTIC restoration (host content preserved, DMC additions removed),
  never over-claimed as byte-identical.
- The dangling-reference scan is scoped to path-like `.md` references over the installed ship-surface
  (24 references covered), asserting no shipped file pulls an unbundled `.md` (DMC-internal
  READMEs/evidence are deliberately NOT shipped).
- The doctor negative controls assert a per-shape exit code (missing interpreter / Ring-0 absent /
  wiring gap ⇒ non-zero), and the Codex forbidden-lexeme control is scoped to `/codex/i` lines only.
