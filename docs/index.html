§

The inviolable loop

Six stages, in order, every substantial change. The critic is never skipped; the verifier never grades its own homework.

01

Plan

Decision-complete plan with explicit scope & acceptance criteria.

02

Critic

A non-authoring reviewer tries to falsify the plan before a line is written.

03

Scope

A compiled scope-lock — edits are refused outside the approved files.

04

Execute

Implementation happens only inside the lock, watched by post-Bash diff guards.

05

Verify

An independent verifier reproduces the result — no self-approval.

06

Evidence

Receipts, verdicts, and a release gate — the record that lets it ship.

LAW No verification, no done. No accepted scope, no edit. No evidence, no completion claim.
§

Enforcement, not etiquette

Guidance is a suggestion an agent can ignore. DMC ships deterministic guards that hold regardless of the model's mood.

Scope-lock

Edits stay in bounds

A compiled lock lists exactly which files may change. Writes outside it — including Bash-mediated ones — are refused.

Release gate

Nine sub-gates to ship

Diff-scope, receipts, findings, goals, decisions, approvals, chain. FAIL dominates; PARTIAL never passes.

Evidence ledger

Every claim has a receipt

Completion is a record, not an assertion — verification refs, minted receipts, an append-only approvals chain.

Secret guard

Never reads your keys

.env, private keys, and credential files are off-limits to every tool, in every mode — by filename, defense-in-depth.

Non-authoring critic

Nothing grades itself

The reviewer and the verifier run in separate lanes from whoever wrote the code. Self-approval is structurally impossible.

Portable control plane

One bin/dmc

A deterministic CLI + hooks that ride along with Claude Code and Codex. No daemon, no lock-in.

Install into any repo

Drop the harness into an existing Claude Code or Codex project. First success is one command; the second command proves it worked.

bash — install DMC
# 1 — get the harness
git clone https://github.com/wjlee930501/Do-Me-Coding
cd Do-Me-Coding

# 2 — install into your project (Claude Code + Codex)
bash .claude/install/dmc-install.sh ~/code/your-repo --host both

# 3 — verify it worked
bin/dmc doctor
doctor: ready · hooks dispatching · mode active
1

Get it

Clone the repo — the whole control plane is bin/, hooks, and .harness/ schemas. No install-time network calls.

2

Adapt it

dmc-install.sh merges hooks & skills into your repo without overwriting your files. Pick --host claude, codex, or both.

3

Prove it

bin/dmc doctor confirms the guards actually dispatch in your host — enforcement you can see, not assume.
