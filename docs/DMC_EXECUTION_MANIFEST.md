# DMC Execution Manifest v2 (v0.3.5)

A **single, forward-looking** artifact that binds everything needed to execute a milestone safely — **task · proposed
provider_target · selected adapter · verification expectations · required human gates · closure criteria** — into one
decision-complete object the loop/human can act on. It **recommends** a binding; it selects/executes/grants nothing.

Generated by `.harness/evidence/dmc-v0.3.5-execution-manifest.sh`, which composes the v0.3.4 provider selector (which
composes the v0.2.8 classifier + v0.2.9 policy + the router). **Additive and read-only** over all of them.

## v2 vs v1 — forward-looking, not a run recorder

| | v1 — `dmc-v0.2.7-run-manifest.sh` | **v2 — `dmc-v0.3.5-execution-manifest.sh`** |
|---|---|---|
| When | **post-hoc** — snapshots a *completed* run | **pre-execution** — binds what *will* execute |
| Contains | commit hash, verify pass/fail counts, origin ahead/behind, push state | task, proposed provider_target, selected adapter, verification *expectations*, gates, closure criteria |
| Git/clock state | yes (HEAD hash, ahead/behind) | **no** (no git hash / ahead-count / wall-clock — deterministic) |

v2 deliberately embeds **no** run state: a forward-looking binding must be deterministic and reproducible from the task
+ policy alone. v1 owns the run-recorder lane; v2 does not duplicate it.

## What the manifest binds

- **task** — `task_id`, `objective`, `context_summary`, and any `provider_target_hint` (recorded, **not** auto-promoted;
  a live-capable hint would require the live gate #5).
- **selection** — the full v0.3.4 selector output (ranked candidates, gates, `stop_and_ask`, `human_gate_required`,
  `fail_closed`).
- **proposed_provider_target** — the **offline-first top-ranked** candidate (`manual_import`, rank 1) with its
  `run_mode` — the safest target. The human/loop may override.
- **selected_adapter** — the adapter the router resolves for the proposed target, obtained via **one** router
  `--print-dispatch` call (the `adapter` field of its JSON). The router answers this **before** any process spawn
  (`provider-router.py:133` returns before `:136`) — it executes nothing.
- **verification_expectations** — the `--verify-script` (or `"verification required"`), `must_pass`, the gate-check
  (stage/commit/push), and the Codex audit (before stage/commit/push). These are **expectations to be met**, not asserted
  as met.
- **required_human_gates** — from the selection (the classifier's gate map).
- **closure_criteria** — the five conditions, as **expectations**: `verified`, `reviewed`, `committed`, `pushed`,
  `closure-recorded` (the handbook / v0.2.9 STOP definition, `docs/DMC_EFFORT_PROVIDER_POLICY.md:48`).

## Gating — three states

| state | trigger | `proposed_provider_target` | `selected_adapter` | `executable_default` | `blocked` |
|---|---|---|---|---|---|
| **low-risk** | docs/test, `human_gate_required=false` | `manual_import` | resolved | **true** | false |
| **gated** | high-risk, `human_gate_required=true` (adapter/protected/live/credential) | `manual_import` (offline-first) | resolved | **false** | false |
| **fail-closed** | classifier absent / selector `fail_closed` / no candidates | **null** | **null** | false | **true** |

A **gated** manifest still proposes the offline-first `manual_import` (the safest target) but marks it
`executable_default:false` — the proposal **requires the human gate** before execution. A live provider is **never**
presented as a no-gate default. Only **fail-closed** nulls the proposal and recommends nothing.

## No execution / no env-secret inference — non-negotiable

- **Executes nothing**: the only router call is the single `--print-dispatch` chokepoint, which hard-codes
  `--print-dispatch` and **never** passes `--live`/`--allow-network`/`--allow-exec`/`--mock`/`--import`. No adapter runs;
  no live/network call.
- **No env/secret inference**: the manifest is a pure function of (task + policy), inherited from the v0.3.4 selector.
  The generator reads **no** env var and **no** `.env*`/credential file; output is byte-identical under `env -i` and across
  a credential-var matrix (proven in the self-test).
- **Free-form task-text redaction** (v0.3.9.1): the task's free-form fields embedded in the manifest (`objective`,
  `context_summary`, `task_id`, `provider_target_hint` — at the top level **and** in the echoed `selection` object) are
  passed through a **value-blind** sanitizer: a token/secret shape becomes `[redacted:unsafe-metadata]` (the matched value
  is **never** re-emitted); safe text is preserved. The sanitizer is a pure regex (no env), so `env -i` byte-identity is
  preserved. Same scanner as the review-packet/closure tools (structured token shapes + sentinels; prefixless
  high-entropy blobs out of scope).
- **Grants no gate**: `side_effects: "none (read-only; executes nothing; grants no gate)"`. The exit code is informational
  and must never be wired to execute.

## Usage

```
dmc-v0.3.5-execution-manifest.sh --task <task.json> [--milestone <id>] [--verify-script <path>] [--out <file>]
dmc-v0.3.5-execution-manifest.sh --self-test
```

`--out` reuses the v0.3.4 hardened guard (refuses protected/secret/symlink targets and **any** `..`-component path,
including a benign-resolving one).
