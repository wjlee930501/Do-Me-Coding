# DMC v1.0 Runtime Architecture

Status: IMPLEMENTED (M2–M9 shipped what this document designs; preserved as the design record —
see docs/DMC_V1_ENFORCEMENT_MATRIX.md for the shipped per-harness reality).
Inputs: `.harness/plans/dmc-v1-runtime-upgrade-audit.md` (Phase 0),
`docs/FABLE_WORKFLOW_TRANSFER.md` (Phase 1, behaviors B1–B13).
Thesis carried forward from v0.6.0: **"Learn suggestions, encode gates."** Nothing below is prompt
text; everything is a deterministic component with schemas, state, and fail modes.

---

## 0. Architectural frame

### 0.1 Three-ring model (the model-independence mechanism)

```
Ring 0  PORTABLE CORE            bin/dmc + lib (POSIX sh + python3), .harness/ state & schemas.
        (harness-agnostic)       All decisions live here: verdict = CLI(args/stdin JSON)
                                 → exit code + JSON verdict. No Claude/Codex coupling.

Ring 1  HARNESS ADAPTERS         adapters/claude-code/ (settings.json + thin hook shims that
        (one dir per harness)    translate hook stdin → Ring-0 CLI call → hook JSON envelope),
                                 adapters/codex/, adapters/opencode/ (v1.0: claude-code full,
                                 codex minimal, opencode stub).

Ring 2  MODEL GUIDANCE           skills, agent prompts, CLAUDE.md/AGENTS.md text. Advisory by
        (per-model packaging)    definition; must never be the only thing standing between the
                                 model and a Ring-0 invariant.
```

Rule R-RING: **any rule enforced only in Ring 2 must be either promoted to Ring 0 or explicitly
labeled `advisory` in its doc.** (Closes audit §3's prose-only gate list.)

### 0.2 Single entry point

`bin/dmc <primitive> <verb> [--json]` — one CLI façade over the existing tools. v1.0 does not
rewrite the shipped v0.2.6–v0.6.5 validators; it **relocates/symlinks them under `bin/lib/` and
routes to them**, preserving their self-tests. (`dmc doctor`, `dmc orient`, `dmc scope lock`,
`dmc verify plan|run`, `dmc gate release`, `dmc worker import`, …). Closes audit blocker B4.

### 0.3 State root

```
.harness/
  repo/          orientation.json, landmarks.json, patterns.json, depsurface.json   (P1–P4)
  runs/<run-id>/ run.json, scope.lock.json, acceptance.json, radius.json,
                 fixloop.log.jsonl, checkpoints.json, decisions.jsonl, receipts/     (P5–P13)
  workers/       tasks/ results/ reviews/ sessions/                                  (P14–P15)
  mode, autonomy-level                                                               (P17)
```

All state is plain JSON/JSONL + the existing schema conventions; every artifact carries the
v0.6.1.0 canonical subject binding (`work_id`, `plan_hash`, `repo_hash`) so the v0.6.5 trace
composes over runs, not just milestones.

### 0.4 Integrity rule for run state

Run-state files are hash-chained (each write appends `prev_hash`) and the **scope/acceptance
files are immutable after plan approval** — the Ring-1 write guards deny agent mutation of
`.harness/runs/<run>/scope.lock.json` and `acceptance.json` (fixes the self-escalation hole,
audit §3). Amendment = new plan revision + re-approval, never in-place edit.

## 1. The twenty primitives

Field key: **Pur** purpose · **In** inputs · **Out** outputs · **St** state ·
**T+** allowed tools · **T−** disallowed · **FS** file surfaces · **Sch** schema requirements ·
**Ix** interactions · **FM** failure modes · **Rec** recovery · **Ver** verification ·
**v1.0** required or deferred.

### P1 — Repository Orientation Primitive  (B1)
- **Pur:** produce/refresh a machine-readable repo map before any planning.
- **In:** worktree, HEAD; optional focus hint.
- **Out:** `repo/orientation.json` {languages, managers, verify_commands, entrypoints, doc_roots,
  head, generated_at}; human summary to stdout.
- **St:** stateless generator; artifact is the state.
- **T+:** read-only FS walk, manifest parsers, `git rev-parse`. **T−:** network, file writes
  outside `repo/`, secret-path reads (delegates path filter to lib/secret-paths).
- **FS:** writes `repo/orientation.json` only.
- **Sch:** new `orientation.schema.md` + validator.
- **Ix:** consumed by P2, P8 (verify commands), P11 (freshness check), plan validator.
- **FM:** stale map (HEAD moved); hallucinated entries if generated by the model instead of the
  tool.
- **Rec:** freshness check refuses stale maps; regeneration is cheap and idempotent.
- **Ver:** self-test on fixture repos (node/python/empty); every emitted path exists; determinism
  at fixed HEAD.
- **v1.0: REQUIRED.**

### P2 — Architecture Landmark Scanner  (B2)
- **Pur:** derive landmark classes {enforcement, contract, release, data, ordinary} replacing
  hand-kept protected lists.
- **In:** orientation.json; built-in heuristics; host overrides (`.harness/repo/landmarks.local.json`).
- **Out:** `repo/landmarks.json` (path → class, reason).
- **St:** artifact-only. **T+:** read-only. **T−:** writes elsewhere, network.
- **Sch:** `landmarks.schema.md`; class enum fixed.
- **Ix:** P5 radius, P7 scope (landmark edits need explicit plan authorization), P16 critic,
  P18 release gate (landmark-diff check).
- **FM:** false negatives (unclassified load-bearing file); silent divergence from legacy lists.
- **Rec:** merge-not-replace migration: v1.0 seeds from the union of existing protected lists,
  flags heuristic disagreements for human triage.
- **Ver:** negative controls (seeded fake hook/schema detected); DMC self-scan must classify its
  own hooks/schemas/adapters as landmarks.
- **v1.0: REQUIRED.**

### P3 — Existing Pattern Extractor  (B3)
- **Pur:** supply exemplar paths per construct class so new code imitates the repo.
- **In:** orientation + query (construct class); **Out:** `repo/patterns.json` entries
  {construct, exemplars[], convention_note}.
- **St:** artifact cache, invalidated on HEAD change. **T+:** read-only search. **T−:** writes.
- **Sch:** `patterns.schema.md` (exemplars must be existing paths).
- **Ix:** plan validator requires an exemplar table for new-construct plans; P16 checks diffs
  against cited exemplars.
- **FM:** exemplar rot; cargo-cult exemplars (wrong analogue). **Rec:** critic finding, re-query.
- **Ver:** exemplar existence check; fixture-repo self-test.
- **v1.0: DEFERRED to v1.1** (schema + plan section land in v1.0; the extractor tool itself is
  the lowest-risk deferral — the plan section can be human/model-filled and validator-checked for
  path existence without the automated extractor).

### P4 — Dependency Surface Scanner  (B5)
- **Pur:** inbound/outbound reference counts for a candidate scope (imports, includes, path
  references).
- **In:** scope candidate list; **Out:** `runs/<run>/depsurface.json` {path → dependents[],
  count}.
- **T+:** read-only grep/AST-lite (regex import scan; no LSP in v1.0). **T−:** writes, network.
- **Sch:** `depsurface.schema.md`.
- **Ix:** feeds P5; cited by plan Change Radius section.
- **FM:** language blind spots (regex misses dynamic imports) — must be labeled `best-effort,
  known-shapes-only` (v0.4.4 honesty pattern).
- **Rec:** unknown-language paths are marked `unscanned`, which P5 treats as radius-unknown ⇒
  escalates review depth.
- **Ver:** fixture repos with known graphs; negative control: a seeded dependent must be found.
- **v1.0: REQUIRED (regex tier only; AST/LSP tier deferred per v0.6.0 defer card).**

### P5 — Change Radius Predictor  (B5, B6)
- **Pur:** turn depsurface + landmarks into a predicted blast list with required checks.
- **In:** depsurface.json, landmarks.json, plan scope. **Out:** `runs/<run>/radius.json`
  {entry: path, dependents, landmark_class, required_check_ids[]}.
- **St:** immutable after plan approval (subject-bound).
- **T+:** pure computation. **T−:** everything else.
- **Sch:** `radius.schema.md`; every entry must reference ≥1 check id in acceptance.json.
- **Ix:** P8/P9 (check linkage), P13 (failure mapping), P16 (critic reviews radius realism).
- **FM:** radius theater (entries without checks) — refused by schema; under-prediction —
  caught post-hoc by P13 and recorded as a finding.
- **Rec:** under-predicted regression ⇒ finding in the v0.6.3 register + radius regenerated.
- **Ver:** validator negative controls; E2E fixture: a change with a known dependent must emit a
  radius entry.
- **v1.0: REQUIRED.**

### P6 — Minimal Diff Planner  (B6)
- **Pur:** make diff bounds explicit and enforceable: plan declares {max_files, max_added,
  max_deleted, forbidden_hunk_classes (e.g. format-churn)}.
- **In:** plan; **Out:** bounds block inside scope.lock.json.
- **Ix:** P7 enforces at runtime; v0.4.3 over-eager guard logic is the enforcement core (promoted
  from advisory to wired).
- **FM:** bounds set absurdly high (defeats purpose) — critic gate reviews bounds vs task class.
- **Rec:** bound hit ⇒ STOP + human decision (raise bound via amendment | revert).
- **Ver:** v0.4.3's existing self-tests (14 PASS) reused after relocation.
- **v1.0: REQUIRED** (mostly relocation + wiring of shipped v0.4.3 logic).

### P7 — Scope Lock Manager  (B4)
- **Pur:** compile approved plan scope → immutable `scope.lock.json`; adjudicate every mutation.
- **In:** approved plan (hash), landmarks. **Out:** scope.lock.json; per-mutation verdicts.
- **St:** the lock file (immutable, §0.4).
- **T+:** none (it *is* the checker invoked by adapters). **T−:** n/a.
- **FS:** verdict path: called by Ring-1 on Edit/Write **and on Bash commands classified as
  write-capable** (new Bash write-radius classifier: redirections, `sed -i`, `tee`, `mv`, `cp`,
  `git apply`, `patch`, interpreter `-c` writes — fail-closed on unparseable commands in active
  mode). Closes audit B1/B2's Bash bypass + `git apply` hole.
- **Sch:** `scope-lock.schema.md` (plan_hash, files[], bounds, immutable stamp).
- **Ix:** P6 bounds; P14 (worker `allowed_files` must be ⊆ run scope at apply time); P18 final
  diff⊆scope check from `git diff --name-only` ground truth.
- **FM:** fail-open on missing interpreter (today's scope-guard); self-edit of the lock.
- **Rec:** `dmc doctor` verifies python3/jq at install; adapters **fail closed in active mode**
  when the checker cannot run (inverting today's default); lock edits denied at Ring 1 + detected
  by hash chain at Ring 0.
- **Ver:** adversarial self-test battery: Bash redirection, symlink, rename, lock-edit attempts
  all denied; legacy scope-guard tests preserved.
- **v1.0: REQUIRED (highest priority).**

### P8 — Acceptance Criteria Compiler  (B7)
- **Pur:** criteria → `acceptance.json` {check_id, kind: command|inspection|human, cmd, expect,
  radius_links[]}; refuse untestable criteria.
- **In:** plan; orientation verify_commands. **Out:** acceptance.json (immutable post-approval).
- **Sch:** `acceptance.schema.md`.
- **Ix:** P9 plans execution order; P10 receipts reference check_ids; P13 counters per check_id.
- **FM:** unverifiable criteria smuggled through as `human` kind — bounded: `human` checks
  require a named question and appear on the release report for the human gate.
- **Rec:** critic REJECT path.
- **Ver:** validator + negative controls (criterion without method refused).
- **v1.0: REQUIRED.**

### P9 — Verification Planner  (B5, B8)
- **Pur:** compute the minimal-sufficient ordered check set for a run (v0.5.5 logic, promoted:
  fed by acceptance.json + radius.json instead of caller-declared facts).
- **In:** acceptance.json, radius.json, task class. **Out:** `runs/<run>/verify-plan.json`.
- **Ix:** P10 executes/records; P18 requires coverage == verify-plan.
- **FM:** under-planning (radius entry with no check) — schema-refused (P5).
- **Ver:** reuse v0.5.5 self-tests (22 PASS) + new linkage tests.
- **v1.0: REQUIRED (promotion of shipped tool).**

### P10 — Evidence Ledger  (B8)
- **Pur:** append-only, subject-bound receipts for every executed check; the only source the
  stop/release gates read.
- **In:** check executions (exit code, bounded redacted output). **Out:**
  `runs/<run>/receipts/*.json` (v0.6.2 evidence-receipt shape) + JSONL index.
- **St:** append-only, hash-chained.
- **T+:** value-blind redactor (v0.5.0 lineage). **T−:** storing secret-shaped values (redact
  known shapes; honest non-completeness attestation per v0.4.4).
- **Sch:** existing `evidence-receipt.schema.md` (+ `check_id` field extension).
- **Ix:** P18 consumes; evidence-log.sh hook becomes a Ring-1 feeder of the ledger instead of an
  unstructured markdown appender.
- **FM:** ledger fragmentation (today's `manual-<timestamp>` per-second files, audit);
  interleaved concurrent writes.
- **Rec:** single JSONL index with O_APPEND line writes; fragments tolerated but indexed.
- **Ver:** v0.6.2 validator (18 PASS) + wiring E2E test (stop blocked without coverage).
- **v1.0: REQUIRED.**

### P11 — Context Recovery Manager  (B11)
- **Pur:** on session start/resume, reconcile declared run state against **observed** git state
  and emit next-safe-action (v0.5.7 logic, promoted from declared-facts to observed-facts).
- **In:** run state files + live `git status`/`diff --name-only`/HEAD. **Out:** resume report;
  next_safe_action enum.
- **Ix:** P12 restore; P17 halt-and-ask on irreconcilable delta.
- **FM:** trusting declared facts (today, `docs/RESUME_RECOVERY.md:88`); double-apply.
- **Rec:** delta ⇒ halt with the diff listed; never auto-reconcile by editing state.
- **Ver:** fixture scenarios: clean-resume, dirty-outside-scope, moved-HEAD, half-applied.
- **v1.0: REQUIRED.**

### P12 — Checkpoint Manager  (B9, B11)
- **Pur:** named known-good points (git ref + state snapshot hash) enabling bounded fix loops to
  revert instead of thrash.
- **In:** run state at green moments. **Out:** `runs/<run>/checkpoints.json`.
- **T+:** `git stash create`/tag-free ref recording (no history mutation). **T−:** any
  history-rewrite; force ops (AUTONOMY always-blocked list).
- **FM:** checkpoint on a false-green (checkpoint requires receipt coverage at creation).
- **Rec:** restore = surfaced instruction to the executor (checkout paths), never automatic
  `reset --hard` (which pre-tool-guard denies).
- **Ver:** fixture: create → break → restore round-trip.
- **v1.0: REQUIRED (minimal ref+snapshot form).**

### P13 — Regression Suspicion Engine  (B9)
- **Pur:** map check failures to radius entries; maintain per-check attempt counters; enforce the
  bounded fix loop.
- **In:** failure events, radius.json, fixloop.log. **Out:** fixloop.log.jsonl; STOP verdict at
  bound.
- **Sch:** `fixloop.schema.md` {check_id, attempt, hypothesis, files_touched}.
- **Ix:** P12 restore suggestion at bound; P17 structured failure report; unpredicted
  regressions ⇒ v0.6.3 finding.
- **FM:** counter reset gaming (new run to clear counters) — counters bind to plan_hash, not
  run-id.
- **Ver:** simulated failing-check fixture; bound triggers STOP; counter survives resume (P11).
- **v1.0: REQUIRED (counters+bound); hypothesis quality is advisory.**

### P14 — Subagent Orchestrator  (B12)
- **Pur:** bind the role taxonomy to actual dispatch: role registry {role → contract: allowed
  tools, artifact schema, may_mutate: bool}, delegation records, artifact validation before
  consumption.
- **In:** decomposition units; role registry. **Out:** `runs/<run>/delegations.jsonl`.
- **T+:** harness subagent APIs via Ring 1. **T−:** dispatching a mutation-capable role without
  scope.lock; nesting beyond declared depth (deterministic depth bound, v0.6.0 defer card
  condition).
- **Sch:** `delegation.schema.md`; role registry file `orchestration/roles.json` — the single
  reconciliation of the 3 drifted taxonomies (audit §11); see
  `docs/DMC_V1_ORCHESTRATION_MODEL.md`.
- **FM:** consuming unvalidated delegate prose; orchestrator self-implementing review roles.
- **Rec:** invalid artifact ⇒ re-dispatch or inline absorption, recorded; role-violation ⇒ stop
  condition.
- **Ver:** schema validation negative controls; E2E dry-run with mock delegates.
- **v1.0: REQUIRED (registry + records + validation); automated multi-agent scheduling deferred.**

### P15 — Worker Proposal Importer  (B12)
- **Pur:** one mechanical gate between a worker/external-model proposal and any repo edit:
  validator PASS + review record PASS + scope compatibility, hash-chained
  task→result→review→apply.
- **In:** task/result/review JSONs. **Out:** import verdict; `apply_authorization.json`
  (consumed by P7 during apply: applied paths must be ⊆ task.allowed_files ∩ run scope).
- **Sch:** existing WORKER_* schemas + new `worker-review` validator (review is 100% prose today,
  audit §3); hardened `worker-result-check.py`: token-class secret patterns (JWT/Bearer/ya29/
  gh[opsu]_ — import oauth-cli's detectors), rename/copy/binary diff parsing, empty-allowed ⇒
  **deny** not skip, task_id/provider cross-check, required-field presence.
- **Ix:** P7 (apply scope), P10 (import receipts), v0.6.1.0 subject binding over worker artifacts.
- **FM:** hand-written result files (provenance) — hash chain makes tampering evident, and the
  importer requires the chain; transcription drift patch→Edit/Write — post-apply diff compared
  against `proposed_patch` paths (fidelity check, names + hunk-count tier in v1.0).
- **Rec:** any REJECT is terminal for that result; re-dispatch with a new task id.
- **Ver:** extend the v0.3.3 contract suite with the empirically-found bypass fixtures (JWT
  result, rename diff, empty-allowed) as permanent regressions.
- **v1.0: REQUIRED.**

### P16 — Critic Gate  (B3, B10)
- **Pur:** adversarial plan/diff review as a *gate with a recorded verdict artifact*, not chat
  prose: verdict JSON {APPROVE|REJECT|NEEDS_CLARIFICATION, blockers[], criteria checked}.
- **In:** plan + orientation/landmarks/radius artifacts. **Out:**
  `runs/<run>/critic-verdict.json`.
- **Sch:** `critic-verdict.schema.md`; C11 invariant: verdict is advisory evidence — it never
  flips approval (approval needs P17).
- **Ix:** plan validator requires a critic verdict ref before start-work; verdict feeds v0.6.5
  trace.
- **FM:** rubber-stamp critic (same context as author) — mitigated by role contract (fresh
  context, read-only tools) in P14 registry.
- **Ver:** schema validator; E2E: start-work refused without verdict artifact.
- **v1.0: REQUIRED.**

### P17 — Human Gate  (B10, B13)
- **Pur:** typed, recorded human authorizations: {plan_approval, scope_amendment, bound_raise,
  release, push, live_call, waiver}; the only source of `approval.source=human-release-gate`
  entries (v0.6.5 Q6).
- **In:** gate requests with context refs. **Out:** approval records (trace-linkage approval
  namespace).
- **St:** `runs/<run>/approvals.jsonl`, append-only.
- **T−:** any inference of approval from critic/verifier/Codex ACCEPT (C11; R12
  anti-laundering).
- **FM:** self-asserted ids (honest-scope debt, Q6 provenance-not-authentication) — v1.0 keeps
  the honest label; cryptographic provenance remains v1.1+ (v0.6.6 mission).
- **Ver:** v0.6.1.0/T7d validator already enforces non-empty auth-id; release gate refuses
  missing approval types.
- **v1.0: REQUIRED (typed records; authentication explicitly out of scope, labeled).**

### P18 — Release Readiness Gate  (B13)
- **Pur:** compose existing validators into one verdict consumed by the stop path and the
  closure flow: diff⊆scope (P7 ground truth) · receipt coverage (v0.6.2) · findings disposition
  (v0.6.3) · goal linkage (v0.6.4) · decision trace (v0.6.5) · gate checks (v0.2.6) · landmark
  diff review flag (P2).
- **In:** full run state. **Out:** `runs/<run>/release-readiness.json` PASS/FAIL per sub-gate.
- **Ix:** Ring-1 Stop adapter calls `dmc gate release --quick` (coverage tier) — replacing the
  keyword-regex stop gate (`stop-verify-gate.sh:64`) with an artifact check that also fires on
  the ultrawork path (run-id creation becomes P7's job, closing the disarmed-gate hole).
- **FM:** gate too slow for a Stop hook — quick tier (state-file checks only, no subprocess
  fan-out) budgeted <2s; full tier for closure.
- **Rec:** FAIL lists gaps; PARTIAL never presented as PASS.
- **Ver:** E2E dry-run fixture (M9): green path passes, each seeded gap fails its sub-gate.
- **v1.0: REQUIRED.**

### P19 — Host Repository Adapter  (B1 applied to hosts)
- **Pur:** installation as *adaptation*: install Ring 0+1, run P1/P2 against the host, generate a
  host profile (verify commands, landmarks, protected merges), self-check.
- **In:** host repo. **Out:** installed surface per a **generated, testable manifest**
  (INSTALL_MANIFEST becomes generated-from-code, killing manifest drift, audit B7);
  `dmc doctor` report (hooks fire, interpreters exist, collision report).
- **Fixes carried:** idempotent CLAUDE.md marker append; uninstaller strip actually implemented
  (gitignore skip-block bug `dmc-uninstall.sh:38-42`); `${DRY:+}` fix; eval-quoting fix;
  host installs receive the Ring-0 control plane (bin/ + schemas) — closing "hosts get v0.1.3
  forever" (audit §4.4).
- **FM:** clobbering another harness — keep the passive-mode auto-detect; doctor verifies
  non-interference.
- **Ver:** install→doctor→uninstall→byte-clean round-trip test on fixture host repos (empty,
  node, existing-claude-settings, existing-OMC).
- **v1.0: REQUIRED.**

### P20 — Model Capability Adapter  (model independence)
- **Pur:** per-model binding profile: `orchestration/models.json` (dated, replaceable, non-load-
  bearing — v0.6.0 lookup pattern) mapping capability classes → available bindings {claude-code
  subagent, worker provider, human}; plus per-harness feature detection (which Ring-1 events
  exist: Codex ships a Stop hook with confirmed `decision:"block"` parity (M6.5 spike, Option A),
  but decision-envelope honoring is unproven turn-free, so the Stop shim is treated as advisory
  and the pre-commit/CI release gate remains the backstop).
- **In:** capability-routing schema (v0.6.1); harness feature matrix. **Out:** binding decisions
  recorded in delegation records.
- **T−:** model-name strings in any gate logic (v0.6.1 self-scan invariant extends to all Ring-0
  code).
- **FM:** silent degradation on a harness lacking an event — must be **loud**: doctor prints the
  enforcement matrix (which invariants are runtime-enforced vs advisory on this harness).
- **Ver:** self-scan (no model names in Ring 0); adapter matrix test per harness fixture.
- **v1.0: REQUIRED (claude-code full matrix; codex matrix = advisory Stop-hook shim +
  pre-commit/CI backstop; opencode stub).**

## 2. Primitive interaction spine (happy path)

```
dmc orient (P1) → dmc landmarks (P2) → plan (+P3 exemplars, P4/P5 radius, P6 bounds, P8 criteria)
  → critic verdict (P16) → human plan approval (P17)
  → dmc run start: scope.lock (P7) + verify-plan (P9) + run-id  [gate now always armed]
  → execute under P7 verdicts (incl. Bash write-radius) · delegate via P14 · import via P15
  → checks → receipts (P10) · failures → fix loop (P13) + checkpoints (P12)
  → resume anytime via P11
  → dmc gate release (P18, quick at Stop / full at closure) → human release approval (P17)
```

## 3. Explicit non-goals for v1.0

- No learned/adaptive gate authority (v0.6.0 anti-goal #8) — all 20 primitives are deterministic.
- No cryptographic approval authentication (P17 labels the debt; v1.1+).
- No LSP/AST dependency analysis (P4 regex tier only; defer card).
- No independent daemon/web UI/MCP server (DMC.md "v1.0 Scope" exclusions stand).
- No automatic patch application from any external model (P15 keeps translate-then-verify).
- No rewrite of shipped validators — relocation + routing + wiring only.
