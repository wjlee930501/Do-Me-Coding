# DMC Review Packet (v0.3.6)

A **read-only** generator that auto-produces a review packet for a milestone — the hand-built review pack, mechanized.
It **records** state for human/external review; it **mutates nothing**, makes **no live call**, and **grants no gate**.

Generated by `.harness/evidence/dmc-v0.3.6-review-packet.sh`.

## The five sections

1. **Changeset summary** — commit hash + subject (or "(staged)"), the `Review-Verdict:` line if present in the message,
   file count + insertions/deletions, and the changed **paths (names only)**.
2. **Protected surfaces touched (review-required)** — changed paths matching the DMC protected-PATH set (the gate-check
   `DEFAULT_PROTECTED` 10 entries), by name. "none" if clean.
3. **Forbidden / secret files (filename-only)** — changed paths matching the CLAUDE.md secret patterns. Any hit ⇒
   **"FORBIDDEN files present — STOP"** + the filenames (contents never read). "none" if clean.
4. **Verification summary** — the `--verify-report`'s **own self-attested** Review-Verdict / count / Final-Status / Run
   ID (advisory; not independently re-verified).
5. **Residual risks** — push state (ahead/behind vs origin/main), protected-surface touches, forbidden-file block,
   auto-log evidence left untracked.

## Secret protection — non-negotiable (names-only; no content; three channels)

The packet **never reads or prints the contents of any secret-bearing file, nor any free-form commit-message body**:

- **File contents** — scans classify by **filename pattern ONLY**; no matched secret file is opened. The changeset is
  read with **names-only git primitives** (`show --name-status`/`--numstat`/`--stat`, `diff --cached --name-status`),
  **never** a content-dumping `git show`/`git diff`, the `-p`/`--patch` family, `git log -p`, `git diff-tree`,
  `git format-patch`, or `git cat-file`.
- **Commit-message body** — commit info uses the **hash (`%H`)** and **subject (`%s`)** only; the `Review-Verdict:` line
  is grepped (single-line, anchored `^Review-Verdict:`). The free-form body (`%b`) is **never** emitted.
- **`--verify-report` path** — a report path matching a secret pattern is **refused unread**.
- **Free-form metadata values** (v0.3.9.1) — every free-form field printed (commit **subject**, the `Review-Verdict:` line,
  the report **Run ID**, the milestone/ref label) is passed through a **value-blind** sanitizer first: if the field
  carries a token/secret shape it is replaced by the deterministic placeholder `[redacted:unsafe-metadata]` (the matched
  value is **never** re-emitted); otherwise it is printed verbatim. Names-only path output is unchanged (paths are the
  deliverable, already run through the forbidden/secret scan). The scanner covers the common **structured** token shapes
  (`sk-`, `AKIA`, PEM `PRIVATE KEY`, `xox*`, `gh[opsu]_`, JWT `eyJ…`, `Bearer …`, `ya29.`, `*_token` assignments) + the
  test sentinels; a **prefixless** high-entropy blob with no recognizable marker is out of scope (entropy heuristics
  false-positive on legit identifiers/hashes).

The self-test proves non-emission with three distinct sentinels (a `.env` file, a commit-message body, and a
secret-pathed `--verify-report`) via `grep -F` over **both** stdout and the `--out` artifact, plus a structural source
audit that **fails** if any content-dumping git primitive (or a non-anchored commit-body format) is reintroduced.

## Secret pattern set (filename only)

`.env*` (except `.example`/`.sample`/`.template`), `*.pem`, `*.key`, `id_rsa`, `id_ed25519`, `*.p12`/`*.pfx`/`*.keystore`,
`credentials`, `.npmrc`, `.netrc`, `.pgpass`, `*service-account*`, `*secret*`, `**/.ssh/*`, `**/.aws/credentials`. The
`credentials` / `*secret*` / `*service-account*` forms intentionally **widen** the CLAUDE.md `*.json` variants
(over-inclusive on the protection side — safe; never a transcription error).

## Usage & exit codes

```
review-packet.sh [--commit <ref>] [--staged] [--repo <dir>] [--verify-report <path>] [--out <file>]
review-packet.sh --self-test
```

- `0` — packet emitted, no forbidden secret file in the changeset.
- `3` — packet emitted, but a **FORBIDDEN** secret file is present in the changeset (advisory STOP; distinct from `2`).
- `2` — usage error / `--out` refused (protected/secret/traversal target; reuses the v0.3.4/v0.3.5 hardened guard incl.
  the `..`-component refusal).

The exit code is **advisory** — it must never be wired to stage/commit/push.
